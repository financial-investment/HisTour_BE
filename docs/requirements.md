# HisTour 요구사항 정의서

## 1. 서비스 개요

HisTour는 사용자가 문화유산 현장에서 사진과 위치 정보를 기반으로 AI 해설을 받고, 탐방 기록을 저장하며, 탐방 종료 후 퀴즈와 리포트로 방문 경험을 복습할 수 있는 문화유산 탐방 기록 서비스이다.

## 2. 서비스 목적

- 문화유산 현장에서 사용자가 현재 보고 있는 대상을 쉽게 이해할 수 있도록 AI 기반 설명을 제공한다.
- 사용자의 실제 탐방 흐름을 여행 단위로 기록하고, 방문한 문화유산 정보를 다시 확인할 수 있게 한다.
- 방문 이력을 기반으로 다음 방문지, 탐방 리포트, 복습 퀴즈를 제공하여 탐방 경험을 확장한다.

## 3. 주요 사용자

| 구분 | 설명 |
| --- | --- |
| 비회원 | 회원가입 및 로그인을 수행할 수 있는 사용자 |
| 회원 | 문화유산 탐방, AI 해설, 추천, 퀴즈, 리포트, 마이페이지 기능을 이용하는 사용자 |
| 운영자 | 문화유산 원천 데이터 및 임베딩 데이터를 적재/관리하는 관리자 |
| 외부 시스템 | 문화유산 공공 API, AI API, 지도/위치 서비스 |

## 5. 비기능 요구사항

| ID | 요구사항 | 설명 |
| --- | --- | --- |
| NFR-SEC-01 | 인증/인가 | JWT 기반 인증을 적용하고, 사용자 소유 리소스 접근을 검증해야 한다. |
| NFR-SEC-02 | 비밀번호 보호 | 사용자 비밀번호는 평문으로 저장하지 않아야 한다. |
| NFR-API-01 | 응답 형식 | API 응답은 `{ success, data, message }` 형식을 일관되게 사용해야 한다. |
| NFR-ERR-01 | 예외 처리 | 인증 실패, 권한 없음, 리소스 없음, 외부 API 실패 등 주요 오류를 구분하여 반환해야 한다. |
| NFR-PERF-01 | 추천 성능 | 문화유산 추천은 Redis Stack 벡터 검색을 활용하여 실시간 탐방 흐름을 방해하지 않아야 한다. |
| NFR-AI-01 | AI 안정성 | AI API 실패 시 사용자에게 실패 원인을 안내하고 서버 장애로 확산되지 않도록 처리해야 한다. |
| NFR-UX-01 | 모바일 UX | 현장 사용을 고려하여 카메라, 위치, 지도, 하단 내비게이션 중심의 모바일 친화 UI를 제공해야 한다. |
| NFR-COMP-01 | 브라우저 호환성 | 위치 정보, 카메라, 지도 SDK가 지원되는 주요 모바일 브라우저에서 정상 동작해야 한다. |
| NFR-MAINT-01 | 문서화 | Swagger UI 또는 API 문서를 통해 주요 API 명세를 확인할 수 있어야 한다. |

## 8. 제약사항 및 가정

- 문화유산 식별은 사용자의 현재 위치와 촬영 이미지를 함께 사용한다.
- 기본 해설 없이 심화 해설만 단독으로 요청할 수 없다.
- 탐방 완료 후 퀴즈와 리포트를 이용하는 흐름을 기본 시나리오로 한다.
- 추천 기능은 문화유산 임베딩 데이터가 Redis Stack에 적재되어 있어야 정상 동작한다.
- 사용자는 본인이 생성한 탐방, 퀴즈, 리포트에만 접근할 수 있다.
